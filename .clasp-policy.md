# Clasp Push Policy

**IMPORTANTE**: Todos los comandos `clasp push` requieren aprobación explícita del usuario.

## Política de Deployment

- ❌ **NO ejecutar** `clasp push` automáticamente
- ✅ **SIEMPRE proponer** el comando y esperar aprobación explícita
- ✅ El usuario debe aprobar explícitamente antes de subir cambios a producción

## Comandos que requieren aprobación

- `clasp push` - Subir cambios a Google Apps Script
- `clasp push --force` - Forzar subida de cambios
- Cualquier comando que modifique el código en producción

## Comandos seguros (no requieren aprobación)

- `clasp pull` - Descargar cambios desde producción
- `clasp status` - Ver estado de sincronización
- `clasp open` - Abrir el proyecto en el editor web
